# Waysheet — План проекта (актуально на 25.04.2026)

## 1) Цель
Кроссплатформенное desktop-приложение для учёта путевых листов с понятным русскоязычным интерфейсом, встроенным управлением БД и быстрым ежедневным вводом данных.

## 2) Текущий статус

### Уже реализовано
- [x] Базовая архитектура C++20 + CMake + Dear ImGui + GLFW + OpenGL3.
- [x] Хранилище SQLite (и stub-режим при отсутствии SQLite).
- [x] Модели данных: `Car`, `Driver`, `Route`, `WaybillEntry`, `WaybillRouteDetail`.
- [x] Создание/открытие/сохранение-копии/закрытие БД.
- [x] Автозагрузка последней использованной БД при старте (`recent_dbs.txt`).
- [x] Современный workspace с вкладками:
  - Дашборд
  - Автомобили
  - Водители
  - Маршруты
  - Путевые листы
  - База данных (в конце списка)
- [x] Единый UX во вкладках справочников и путевых листов:
  - верхняя панель действий,
  - слева список,
  - справа редактор выбранной записи.
- [x] Убрана отдельная кнопка «Редактировать»: редактирование сразу в правой панели.
- [x] Автосохранение изменений выделенной записи (cars/drivers/routes/waybills).
- [x] В списках после добавления автоматически выделяется новая запись + фокус на первое поле редактора.
- [x] В путевом листе реализован список участков/маршрутов в том же стиле (слева список, справа редактор).
- [x] Добавлен календарь для выбора даты путевого листа.
- [x] Добавлены начальные остатки автомобиля и автоматический расчет начала/конца путевого листа по предыдущему листу.
- [x] Добавлены нормы расхода по трассам в разрезе каждого автомобиля.
- [x] Упрощен справочник маршрутов: нормы убраны из вкладки маршрутов и ведутся только в карточке автомобиля.
- [x] Новый путевой лист создается с сегодняшней датой, пустыми участками и нулевой заправкой.
- [x] Добавлен каскадный пересчет путевых листов по автомобилю в хронологии дата + ID при исправлениях внутри периода.
- [x] Добавлена вкладка итогов за месяц со сводкой по автомобилям и водителям.
- [x] Русификация интерфейса.
- [x] Локальная загрузка шрифтов из `fonts/`:
  - `NotoSans-Regular.ttf` (кириллица),
  - Font Awesome (`fa-solid-900.ttf` / `fontawesome-webfont.ttf`) для иконок.
- [x] Копирование шрифтов в каталог сборки (`build/Release/fonts`) через CMake post-build.

### Что ещё не закрыто полностью

- [ ] Полноценная аналитика на дашборде (графики, срезы по периодам).
- [ ] Расширенные групповые операции и массовые действия.

## 3) Архитектурные ориентиры (без ломки текущего UX)
- Сохраняем pattern: `Toolbar -> Filters/Group actions -> List (left) -> Editor (right)`.
- Любое изменение данных — через автосохранение, без лишних модальных шагов.
- Любое добавление — с немедленным переходом в редактирование новой записи.
- Все текстовые и иконковые glyphs — из локальных шрифтов в каталоге проекта/сборки.

## 4) План следующих итераций

### Итерация A — Печать и экспорт
1. [x] Экспорт журнала путевых листов в CSV.
2. Экспорт выбранного путевого листа в печатный формат.
3. Единые кнопки экспорта в верхних панелях вкладок.

### Итерация B — Дашборд и аналитика
1. KPI за выбранный период: пробег, норматив, факт, отклонение.
2. Топ маршрутов и машин по пробегу/перерасходу.
3. Мини-графики по дням.

### Итерация C — Качество и стабильность
1. Проверки валидации дат/чисел/обязательных полей.
2. Регрессионные тест-сценарии для CRUD и автосохранения.
3. Улучшение сообщений об ошибках БД и UX при отказах операций.

## 5) Критерии готовности MVP+
- Пользователь запускает приложение и сразу работает с последней БД.
- Любая запись (справочники + путевые листы) добавляется и редактируется без «пустых» кликов.
- Интерфейс русскоязычный, шрифты и иконки стабильно отображаются на чистой машине.
- Базовые отчётные операции (минимум CSV/печать) доступны из UI.
